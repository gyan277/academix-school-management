# Automatic Student & Staff Number Generation

## Overview
Implemented database triggers to automatically generate unique student numbers and staff numbers when new records are added.

---

## Problem
When adding a new student or staff member, the `student_number` or `staff_number` field was not being populated automatically, leaving it empty.

## Solution
Created PostgreSQL triggers that automatically generate sequential numbers based on the school prefix.

---

## Student Number Format

**Format**: `[SCHOOL_PREFIX][NUMBER]`

**Examples**:
- `MOU0001` - First student at Mount Olivet Methodist Academy
- `MOU0002` - Second student
- `MOU0003` - Third student

**Components**:
- **SCHOOL_PREFIX**: First 3 letters of school name (e.g., "MOU" for "Mount Olivet")
- **NUMBER**: Sequential 4-digit number (0001, 0002, 0003, etc.)

---

## Staff Number Format

**Format**: `[SCHOOL_PREFIX]-S[NUMBER]`

**Examples**:
- `MOU-S001` - First staff member at Mount Olivet Methodist Academy
- `MOU-S002` - Second staff member
- `MOU-S003` - Third staff member

**Components**:
- **SCHOOL_PREFIX**: First 3 letters of school name
- **-S**: Indicates staff member
- **NUMBER**: Sequential 3-digit number (001, 002, 003, etc.)

---

## How It Works

### Database Triggers

#### Student Number Trigger
1. Fires **before** a new student is inserted
2. Checks if `student_number` is NULL or empty
3. Gets the school prefix from `school_settings` table
4. Finds the highest existing number for that school
5. Generates the next sequential number
6. Assigns it to the new student record

#### Staff Number Trigger
1. Fires **before** a new staff member is inserted
2. Checks if `staff_number` is NULL or empty
3. Gets the school prefix from `school_settings` table
4. Finds the highest existing number for that school
5. Generates the next sequential number with "-S" prefix
6. Assigns it to the new staff record

### Frontend Changes

**Before**:
```typescript
// Manual generation (unreliable)
const count = students.length + 1;
const studentId = `STU${String(count).padStart(4, "0")}`;

await supabase.from("students").insert([{
  student_id: studentId,  // Manually generated
  full_name: newStudent.name,
  // ... other fields
}]);
```

**After**:
```typescript
// Database handles it automatically
await supabase.from("students").insert([{
  // No student_number field needed!
  full_name: newStudent.name,
  // ... other fields
}]);
// student_number is automatically generated by trigger
```

## Verification Queries

### Check Student Numbers
```sql
-- View all students with their numbers
SELECT 
  student_number,
  full_name,
  class,
  admission_date
FROM public.students
ORDER BY student_number;

-- Count students by school
SELECT 
  ss.name as school_name,
  COUNT(*) as total_students,
  COUNT(CASE WHEN s.student_number IS NOT NULL THEN 1 END) as with_number,
  COUNT(CASE WHEN s.student_number IS NULL THEN 1 END) as without_number
FROM public.students s
LEFT JOIN public.school_settings ss ON s.school_id = ss.id
GROUP BY ss.name;
```

## Troubleshooting

### Issue: Student number still empty after adding student

**Solution**:
1. Check if trigger is installed:
```sql
SELECT * FROM information_schema.triggers 
WHERE trigger_name = 'generate_student_number_trigger';
```

2. If not found, run `AUTO_GENERATE_STUDENT_NUMBER.sql`

3. Manually update existing students:
```sql
-- This is included in the SQL file
-- It will generate numbers for all students without one
```

### Issue: Numbers not sequential

**Reason**: Students were added before trigger was installed

**Solution**: Run the update script in `AUTO_GENERATE_STUDENT_NUMBER.sql` which will renumber all students sequentially based on creation date.

### Issue: Wrong school prefix

**Check school name**:
```sql
SELECT id, name, UPPER(SUBSTRING(name FROM 1 FOR 3)) as prefix
FROM public.school_settings;
```

The prefix is the first 3 letters of the school name.
